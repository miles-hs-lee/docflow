# DocFlow 사용 설명서

문서를 **안전하게 공유**하고, **누가 · 언제 · 어디까지 봤는지** 확인할 수 있는 서비스입니다.
투자 자료, 영업 제안서, 계약서처럼 "그냥 이메일 첨부로 보내기엔 신경 쓰이는" 문서를 보낼 때 유용합니다.

> 이 설명서는 기술 지식이 없어도 따라 할 수 있도록 쓰였습니다. 시스템 자동 연동(개발자용)은 맨 아래 [개발자·연동](#개발자--연동) 참고.

## DocFlow로 무엇을 할 수 있나요?

- 📄 **PDF 문서를 링크로 공유** — 받는 사람은 **가입할 필요 없이** 링크만 열면 됩니다.
- 🔒 **열람 조건 설정** — 비밀번호, 이메일 입력, 비밀유지(NDA) 동의, 만료일, 다운로드 금지, 워터마크 등.
- 📊 **열람 분석** — 누가 몇 번, 어느 페이지까지 봤는지 확인.
- 🗂 **데이터룸** — 여러 문서를 폴더로 묶어 하나의 자료실로 공유.
- 📥 **파일 요청** — 반대로 상대방이 나에게 파일을 올리도록 요청.
- 🎨 **내 브랜드 적용** — 로고·색상·회사명으로 화이트라벨링.
- 👥 **팀 협업** — 동료를 초대해 함께 관리.

## 시작하기

1. **회원가입 / 로그인** — 이메일과 비밀번호로 가입합니다. 로그인하면 **대시보드**로 들어갑니다.
2. **화면 둘러보기** — 상단의 메뉴 탭으로 모든 기능에 접근합니다:

| 메뉴 | 하는 일 |
|---|---|
| **대시보드** | 전체 현황 한눈에 (열람 수·인기 문서·최근 활동) |
| **콘텐츠** | 문서(PDF) 업로드 + 공유 링크 관리 |
| **데이터룸** | 여러 문서를 묶은 자료실 만들기 |
| **연락처** | 문서를 본 방문자(이메일) 모아보기 |
| **파일요청** | 상대방이 나에게 파일을 올리도록 요청 |
| **자동화** | 열람·수신 시 Teams 알림 받기 |
| **팀** | 동료 초대·역할 관리 |
| **휴지통** | 삭제한 항목 복구 |
| **설정** | 로고·브랜드 색상 등 브랜딩 |

> 화면 상단에서 **워크스페이스(팀 공간)를 전환**할 수 있습니다. 자세한 내용은 [팀으로 함께 쓰기](#7-팀으로-함께-쓰기).

## 1. 문서 올리기 (콘텐츠)

1. **콘텐츠** 메뉴로 이동합니다.
2. **파일 업로드**를 눌러 PDF 파일을 올립니다. (현재 **PDF** 형식을 지원합니다.)
3. 업로드한 문서는 목록에 쌓이고, 여기서 바로 공유 링크를 만들 수 있습니다.

---

## 2. 공유 링크 만들기

문서를 안전하게 보내는 핵심 기능입니다.

1. **콘텐츠**에서 공유할 문서를 고르고 **공유 링크 만들기**를 누릅니다.
2. 아래 **열람 조건**을 원하는 대로 설정합니다(전부 선택 사항):

| 옵션 | 설명 |
|---|---|
| **이메일 입력 요구** | 문서를 열기 전에 방문자가 이메일을 입력해야 합니다 → 누가 봤는지 식별됩니다. |
| **허용 이메일 도메인** | 특정 회사 도메인(예: `@acme.com`)만 열람 허용. |
| **비밀번호** | 비밀번호를 아는 사람만 열람. |
| **비밀유지(NDA) 동의** | 열람 전 동의 문구에 체크해야 합니다. |
| **만료일** | 지정한 날짜 이후 자동으로 막힙니다. |
| **최대 열람 횟수** | 정해진 횟수만큼만 열람 가능. |
| **1회용** | 한 번 열람하면 만료. |
| **다운로드 허용/금지** | 다운로드 버튼 노출 여부 (기본: 금지). |
| **워터마크** | 화면에 방문자 정보를 흐리게 겹쳐 유출을 억제. |

3. 만들어진 링크(`.../v/...`)를 복사해 메일·메신저로 전달하면 됩니다.
4. 링크는 언제든 **수정**(조건 변경)하거나 **비활성화**할 수 있습니다 — 이미 보낸 링크에도 즉시 반영됩니다.

> 💡 받는 사람은 **가입·로그인 없이** 링크만 열면 됩니다.

---

## 3. 누가 봤는지 확인하기 (분석)

- **대시보드**: 전체 열람 수, 순방문자, 인기 문서, 최근 활동을 한눈에 봅니다.
- **링크/문서별 통계**: 열람 횟수, 순방문자 수, 다운로드 수, **페이지별로 얼마나 머물렀는지**까지 확인합니다.
- **연락처**: 이메일을 입력하고 문서를 본 방문자들이 한곳에 모입니다. 어떤 문서를 봤는지도 함께 보이며, 필요하면 목록을 **내보내기(CSV)** 할 수 있습니다.

> 방문자를 정확히 식별하려면 링크에 **이메일 입력 요구**를 켜두세요.

## 4. 데이터룸 — 여러 문서를 한 번에

여러 PDF를 **폴더로 정리해 하나의 링크로** 공유하는 자료실입니다. 투자 실사 자료, 계약 패키지 등에 적합합니다.

1. **데이터룸** 메뉴에서 새 데이터룸을 **이름만으로** 만듭니다.
2. 방 안에서 **파일을 추가**하고, **폴더**로 정리하고, **순서**를 바꿉니다.
   - "제거"는 방에서 빼는 것일 뿐, 원본 문서와 다른 방에는 영향이 없습니다.
3. **접근 권한(뷰어 그룹)** — 받는 사람마다 볼 수 있는 폴더를 다르게 줄 수 있습니다. (예: A사에는 1·2폴더만, B사에는 전체)
4. **Q&A** — 방문자가 자료를 보다 질문을 남기면, 방 화면에서 답변할 수 있습니다. (방문자는 자기 질문만 봅니다.)
5. **데이터룸별 브랜딩** — 이 방에만 적용되는 로고·색상·커버 이미지를 따로 둘 수 있습니다.

데이터룸도 일반 문서와 똑같이 **공유 링크 + 열람 조건 + 분석**을 그대로 사용합니다.

## 5. 파일 받기 (파일 요청)

공유와 **반대 방향** — 상대방이 **나에게** 파일을 올리도록 요청합니다.

1. **파일요청** 메뉴에서 요청을 만듭니다(제목·안내문).
2. 옵션: **이메일 입력 요구**, **최대 업로드 수**, **만료일**.
3. 생성된 요청 링크(`.../r/...`)를 전달하면, 상대는 그 페이지에서 파일을 업로드합니다.
4. 받은 파일은 **파일요청** 메뉴에서 확인·다운로드합니다.

> 받은 파일은 내 일반 문서 보관함과 **분리**되어 안전하게 관리됩니다.

---

## 6. 내 브랜드로 꾸미기 (브랜딩)

공유 화면을 내 회사처럼 보이게 만듭니다.

1. **설정** 메뉴 → **브랜딩**으로 이동합니다.
2. 설정할 수 있는 항목: **회사명**, **브랜드 색상**(색상 선택기 또는 HEX 입력), **로고**, **커버 이미지**(공유 화면 상단 배너).
3. 하나라도 설정하면, 공유 링크·파일요청 화면에서 **DocFlow 표기가 사라지고 내 브랜드만** 표시됩니다. 아무것도 설정하지 않으면 기본 DocFlow 디자인이 적용됩니다.

> 데이터룸은 방마다 별도 브랜딩을 둘 수 있고, 비워둔 항목은 계정 브랜딩을 물려받습니다.

---

## 7. 팀으로 함께 쓰기

**워크스페이스**는 하나의 팀 공간입니다. 그 안의 문서·링크·분석을 팀원과 함께 관리합니다.

- **여러 워크스페이스**를 만들고 상단에서 **전환**할 수 있습니다. (예: 회사용 / 개인용)
- **팀** 메뉴에서 **동료를 초대**합니다 — 초대 링크를 만들어 전달하면, 상대가 그 링크로 합류합니다.
- **역할**:
  | 역할 | 권한 |
  |---|---|
  | **소유자(owner)** | 모든 권한 + 다른 소유자 지정/제거 |
  | **관리자(admin)** | 멤버 초대·관리, 콘텐츠 관리 |
  | **멤버(member)** | 콘텐츠·링크 작업 |

> 새로 가입하면 **개인 워크스페이스**가 자동으로 만들어집니다.

## 8. 알림 받기 (자동화)

문서가 열렸을 때, 파일을 받았을 때 등 주요 순간에 **Microsoft Teams로 실시간 알림**을 받을 수 있습니다.

1. **자동화** 메뉴로 이동합니다.
2. **Teams Incoming Webhook 주소**를 등록하고, 받고 싶은 **이벤트**를 선택합니다.
   - 예: 열람 성공, 다운로드, 접근 거부, 파일 수신, 데이터룸 질문, 멤버 합류 등.
3. 저장하면 해당 이벤트가 일어날 때마다 Teams 채널로 카드 알림이 도착합니다.

> 더 고급 연동(다른 도구로 자동 전송, 시스템 연동)은 [개발자·연동](#개발자--연동)을 참고하세요.

---

## 9. 정리하기 — 휴지통 · 계정

- **휴지통**: 삭제한 링크·문서는 휴지통으로 가며, 필요하면 **복구**할 수 있습니다.
- **계정 삭제**: 설정에서 계정을 삭제하면 내 문서·링크·업로드가 함께 정리됩니다. (팀의 유일한 소유자라면, 먼저 다른 소유자를 지정해야 안전하게 나갈 수 있습니다.)

## 자주 묻는 질문 (FAQ)

**Q. 어떤 파일을 올릴 수 있나요?**
현재 **PDF** 문서를 지원합니다.

**Q. 받는 사람도 가입해야 하나요?**
아니요. 받는 사람은 **링크만 열면** 됩니다. (단, 링크에 비밀번호·이메일 입력 등을 걸었다면 그 조건은 충족해야 합니다.)

**Q. 이미 보낸 링크를 막을 수 있나요?**
네. 링크를 **비활성화**하거나 **휴지통**으로 보내면 즉시 막힙니다.

**Q. 누가 봤는지 정확히 알 수 있나요?**
링크에 **이메일 입력 요구**를 켜두면, 문서를 본 사람의 이메일이 **연락처**에 기록됩니다.

**Q. 다운로드를 막을 수 있나요?**
네. 링크 설정에서 **다운로드 금지**(기본값)로 두면 화면 열람만 가능합니다. **워터마크**도 함께 쓰면 유출 위험을 더 줄일 수 있습니다.

**Q. 한 번에 여러 문서를 보내려면?**
**데이터룸**으로 여러 문서를 묶어 하나의 링크로 공유하세요.
